# 🎨 Foundation Design System

Отлично. Теперь мы переходим к этапу, который в крупных продуктовых компаниях называют **Foundation Design System**. Это фундамент всего приложения. После его создания любой новый экран собирается как из конструктора LEGO.

Я предлагаю строить дизайн в стиле **Material 3 + Apple Human Interface Guidelines + современный минимализм**, но с собственным фирменным стилем **Кала-Такси**.

# 🚖 Master Screen — Home + Map + Bottom Sheet

Согласен. В современных сервисах такси (**Uber**, **Bolt**, **Яндекс Go**, **DiDi**) именно главный экран является ядром продукта. Если он спроектирован правильно, то около **70% интерфейса уже готово**.

Я бы пошел еще глубже и сначала разработал **Master Screen** — экран, из которого логически вырастают почти все остальные состояния приложения.

Вместо двух отдельных полей предлагаю единый блок маршрута.

```text
╭────────────────────────────╮
│ ○ Откуда                   │
│ │                          │
│ ● Куда                     │
╰────────────────────────────╯
```

### Преимущества

* занимает меньше места;
* маршрут воспринимается как единый объект;
* проще масштабировать для добавления промежуточных остановок.

# 🚕

## Карточка тарифа

Каждый тариф представлен как отдельная карточка.

```text
╭────────────────────────────╮
│ 🚕 Эконом                  │
│                            │
│ Через 4 минуты             │
│                            │
│ 1200 ₸                     │
╰────────────────────────────╯
```

При выборе карточка:

* увеличивается на **2–3%**;
* получает зеленую рамку;
* показывает галочку.

# 🧭 Навигация

Нижняя навигация должна появляться только на главных разделах приложения.

|    🏠   |    🕘   |   💳   |    👤   |
| :-----: | :-----: | :----: | :-----: |
| Главная | История | Оплата | Профиль |

Во время активного заказа она скрывается, чтобы освободить место для карты и панели поездки.

---

# ✨ Микроанимации

Для ощущения «живого» интерфейса стоит использовать последовательные, короткие анимации.

* Bottom Sheet плавно раскрывается и фиксируется в трех состояниях.
* Кнопки слегка увеличиваются при нажатии (около **98–99% масштаба**).
* Карточки тарифов плавно подсвечиваются при выборе.
* Маркеры на карте появляются с легким масштабированием.
* При назначении водителя карточка выезжает снизу, а карта автоматически центрируется на маршруте.

# ➜ Что делать дальше

Перед созданием остальных экранов я бы спроектировал еще **5 ключевых компонентов**, которые будут переиспользоваться во всем приложении.

1. **Bottom Sheet** — единый контейнер с тремя уровнями раскрытия и поддержкой жестов.
2. **Ride Card** — универсальная карточка для тарифа, водителя, истории и подтверждения.
3. **Map Controls** — плавающие кнопки управления картой (местоположение, слои, масштаб).
4. **Search & Route Component** — единый компонент для выбора маршрута с возможностью добавлять промежуточные точки.
5. **Navigation System** — нижняя навигация и правила ее поведения в обычном режиме, при заказе и во время поездки.

После этого остальные **20 экранов** действительно можно будет собрать как конструктор из уже готовых компонентов, без создания уникального интерфейса для каждого сценария. Это обеспечит единообразие, ускорит разработку и упростит дальнейшее масштабирование продукта.
